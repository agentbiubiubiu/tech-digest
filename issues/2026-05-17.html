🔥 头条深读

本周，三家基础设施公司几乎同时交出了对「Agent 该在哪里跑代码」的不同答卷。Fly.io 发文直言：只读的 ephemeral sandbox 是对 Agent 的结构性浪费——依赖安装、跨 PR 状态保留、日志读取、持续迭代，每一步都在重复造轮子。他们推出 Sprites 持久云电脑，配合 checkpoint/restore 和常驻文件系统，让 Agent 以更接近人类工程师的方式工作。

Modal 则从另一个角度切入：发布编码 Agent 沙箱横评，对比 gVisor、Firecracker 等隔离技术在 GPU 能力、并发与冷启动上的差异，并用实际客户案例（Lovable 促销周末 2 万并发沙箱）证明其弹性。LangChain 紧随其后推出 LangSmith Sandboxes，支持自定义镜像、长会话状态保留、Auth Proxy 凭证隔离——把安全沙箱与追踪/部署/评测深度耦合。

与此同时，Vercel 发布 AI Gateway 生产指数，基于七个月、跨数百模型与数十万亿 token 的真实流量数据，揭示 Agent 工作负载已占据生产流量的显著比例。这不再是实验室里的技术选型问题——当 Agent 成为生产系统的主要消费者，谁控制了执行环境、谁控制了可观测性、谁控制了安全边界，谁就控制了 AI 工程的下一个利润池。

我的判断：持久云电脑与安全沙箱不是非此即彼，而是会沿着信任边界分化——内部可信 Agent 用持久环境提效，外部/untrusted Agent 用微虚拟机隔离风险。真正的赢家是能同时提供两者、并用统一的可观测性层串起来的平台。这场基础设施之争的胜负，将决定未来三年 Agent 工程的成本结构与锁定程度。

确定性：confirmed（产品均已公开发布或进入公测）

🧠 今日认知升级

认知升级 1：执行环境是新的锁定层。过去十年，锁定发生在云服务（AWS/GCP）、容器编排（K8s）、CI/CD（GitHub Actions）层面。现在，Agent 运行时正在成为新的黏性层——一旦你的 Agent 依赖了某家的文件系统、凭证注入、网络隧道和状态 checkpoint，迁移成本将比换一个模型 API 高出一个数量级。

认知升级 2：开放模型的「够用门槛」正在从聊天场景向 Agent 场景迁移。Interconnects 月度开放模型综述显示，Gemma 4、DeepSeek V4 等开放权重在代码生成、工具调用、长上下文等 Agent 核心能力上已接近封闭旗舰。这意味着「必须用 Claude/GPT API 才能跑 Agent」的假设正在失效，而这将彻底改变 Agent 平台的成本模型与商业谈判筹码。

认知升级 3：可观测性正在从「看指标」演变为「驱动闭环」。LangSmith Engine 不只是看 trace——它聚类失败、连接代码库做根因解释、自动开 PR、同步生成评测数据集。Datadog 在截图里嵌入隐写水印让截图可反查原始查询。这意味着可观测性不再是被动的「出了事看 dashboard」，而是主动的「数据驱动持续改进飞轮」。这套能力对 Agent 尤其关键，因为 Agent 的失败模式远比传统软件更隐蔽和多样。

认知升级 4：工程纪律的外化是 Agent 时代的必需品。Addy Osmani 的「Agent Skills」文章和美团 31 万行 AI 重构实践共同指向一个结论：当 90% 代码由 AI 生成时，靠个人代码审查已不可能维持质量。你必须把工程纪律——架构约束、测试覆盖、安全检查——编码成 Agent 不可绕过的执行规则。这不是工具选型问题，而是组织能力重构。

CTOAgent 运行时正在成为新的平台锁定层。需要在内部建立统一的 Agent 执行抽象，避免被单一供应商绑定。同时，开放模型性能逼近封闭旗舰意味着多模型策略从成本优化升级为战略必选。执行环境的统一抽象比模型选择更重要——后者可以随时切换，前者一旦耦合就很难迁移理解平台锁定从「云服务」到「Agent 运行时」的迁移路径，提前建立可替换架构

创业者Agent 执行环境的碎片化意味着有机会做「统一抽象层」——像 Terraform 统一多云一样统一多 Agent 运行时。或者反过来，在某个垂直领域做深度集成的 Agent + 运行时一体化产品。MVP 可以从一个痛点场景切入：比如「给 QA 团队的持久化测试 Agent 环境」。做 Agent 运行时的统一抽象，或做垂直场景的 Agent + 环境一体化——两条路都有机会掌握「基础设施碎片化 → 统一抽象层」的创业模式识别能力
思想者 / 写作者Agent 运行时之争揭示了一个更深层的命题：当 AI 代理需要「像人一样工作」——有持久的工作空间、记忆、工具和环境——我们实际上在为软件创造一种新的「栖息地」。这改变了人与工具的关系：工具不再是被动调用的函数，而是需要被「安置」的实体。这个思想转变值得写成一篇关于「AI 栖息地经济学」的长文。从「调用 API」到「安置 Agent」的转变，是人机关系的一次范式迁移「AI 栖息地」这个隐喻可以成为理解未来十年 AI 基础设施演进的核心框架

🎯 身份跃迁训练

如果我是 CTO，我今天应该学到什么？

  • Agent 运行时碎片化暴露了平台工程的新挑战：执行环境、凭证管理、网络隔离、状态持久化——这些能力应该由平台团队统一提供，还是让各业务团队自行选择？答案是平台化：建立统一的 Agent 执行抽象层，像内部 PaaS 一样提供标准化的 Agent 运行环境。
  • 开放模型性能逼近封闭旗舰，意味着 CTO 需要重新评估模型采购策略。不是「选最好的」，而是「选最可替换的」——把模型当商品而不是合作伙伴。这需要在架构上做到模型无关。
  • Agent 工程纪律的外化（Skills、Rules、Hooks）本质上是在为 AI 建立「组织制度」。CTO 应该把这视为与人类工程制度同等重要的能力建设，而不是技术细节。
  • 可观测性从「看 dashboard」到「驱动闭环」的跳跃，对技术组织意味着 SRE 团队需要新技能：不仅要会设警报，还要会构建自动化修复流水线。这是组织能力，不只是工具采购。

如果我是 CEO，我今天应该学到什么？

  • AI 价值链重组意味着商业模式需要重新定位。如果你的产品只是「套壳 API」，利润会被上游模型商和下游运行时平台挤压。需要找到自己的「不可替代层」——可能是行业数据、可能是工作流知识、可能是合规能力。
  • Vercel 的生产数据显示 Agent 流量占主导——这意味着 B2B 客户的采购决策正在从「买什么模型」转向「买什么 Agent 平台」。如果你在做 toB 产品，需要围绕这个新的采购决策点重构销售叙事。
  • 开放模型崛起对定价权的影响是结构性的：当客户有多个「够用」的选择时，你无法靠模型能力溢价。增长杠杆要从「模型更强」转向「工作流更完整、切换成本更高」。
  • 美团 31 万行 AI 重构的数据表明，AI 编码在大厂已经是现实而非试验。CEO 需要关注的不是「要不要用 AI 编码」，而是「AI 编码的质量治理体系是否跟上了产出速度」。
  • Agent 执行环境的选择会影响企业的安全态势和合规边界。CEO 需要确保安全团队参与 Agent 运行时的选型，而不是事后补救。

如果我是投资人，我今天应该学到什么？

  • Agent 运行时赛道处于早期，但已出现明确的分化信号：Fly.io 走「持久云电脑」路线，Modal 走「弹性沙箱」路线，LangChain 走「平台内嵌」路线。投资者应关注哪种模式的黏性更强——持久环境天然有更高的迁移成本。
  • 开放模型的密集发布正在改变 AI 投资的风险结构：押注单一闭源模型公司的护城河正在被侵蚀。更安全的投资方向可能是模型无关的基础设施层——Gateway、执行环境、可观测性。
  • SemiAnalysis 的价值链分析显示 GPU 集群 TCO 与模型实验室的利润捕获在同步增长。投资者需要区分「模型层利润」和「基础设施层利润」——前者面临竞争侵蚀，后者可能有更持久的定价权。
  • 最大风险是「Agent 泡沫」——如果 Agent 的实际 ROI 无法覆盖运行时基础设施的成本，整个赛道可能面临重新定价。需要看到像 Lovable 这样的真实客户案例和 Vercel 这样的生产数据来验证需求。
  • 护城河在哪里？数据飞轮（LangSmith 的 traces → 改进 → 评测循环）可能比技术壁垒更重要。投资应关注谁能最快把客户数据转化为产品改进。

如果我是创业者，我今天应该学到什么？

  • Agent 运行时碎片化 = 创业机会。三个切入点：(1) 跨运行时的统一抽象层；(2) 特定行业的 Agent + 环境一体化（如医疗合规 Agent 环境）；(3) Agent 运行时的可观测性与安全审计工具。
  • MVP 建议：从一个高痛点场景开始，比如「给 QA 团队的持久化测试 Agent」或「给合规团队的审计 Agent 沙箱」。不要一上来就做通用平台。
  • 第一批用户是谁？已经在用 Agent 编码但被沙箱限制困扰的工程团队——他们每天都在重复安装依赖、重建环境、丢失状态。这个痛点是真实且频繁的。
  • 最大风险：大平台（AWS、GCP、Azure）可能会内建 Agent 运行时能力。防御策略是在垂直场景做深度集成、积累行业数据和工作流知识。
  • 开放模型崛起为创业者提供了新的成本优势：不再需要为模型 API 付费，可以用开放模型 + 自建运行时提供比 SaaS Agent 更低成本的解决方案。

如果我是思想者 / 写作者，我今天应该学到什么？

  • 「给 Agent 一台真机」这个思路，本质上是在为软件创造一种新的存在形式——不再是无状态的函数调用，而是有「住所」、有「记忆」、有「工具箱」的持久实体。这改变了我们对软件本体论的理解。
  • Agent 运行时之争也是一场关于「信任」的哲学辩论：我们信任 Agent 到什么程度？给它持久环境意味着更大的信任（和更大的风险），给它临时沙箱意味着更小的信任（和更多的摩擦）。信任的边界在哪里？
  • 一句值得记住的表达：「Fly.io 说得对——只读沙箱是对 Agent 的结构性歧视。但 Modal 也说得对——不可信代码需要隔离。真正的问题不是选哪个，而是如何在同一个平台上同时提供两者。」
  • 这件事背后反映了人与工具关系的一次范式迁移：从「人使用工具」到「人为工具提供栖息地」。这个转变可以写成一篇关于「AI 栖息地经济学」的长文，探讨当 Agent 需要「生活空间」时，会如何改变计算基础设施的经济学和所有权结构。
