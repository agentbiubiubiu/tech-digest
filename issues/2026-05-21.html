🔥 头条深读

OpenAI 内部通用推理模型在未经专项数学脚手架的情况下，自主证伪了 Erdős 于 1946 年提出的平面单位距离问题核心猜想。模型构造出无限族点集配置，证明下界可突破 n1+o(1)，推翻沿用近 80 年的主流预期。

证明已由 Noga Alon、Tim Gowers 等数学家审阅确认。Noga Alon 评价：「这是 AI 首次自主解决组合几何领域的核心开放问题。」令人意外的是，模型在证明中引入了代数数论工具来解决初等几何问题——这种跨领域迁移正是人类创造性研究的标志性行为。

这一里程碑的意义不仅在于单个定理。它表明前沿推理模型已具备三重能力：原创性洞察（而非暴力搜索）、长链严格证明、以及跨学科工具迁移。此前 AlphaProof、AlphaGeometry 等系统需要大量领域特化与脚手架，而此次使用的是通用模型通用推理。

对团队的启发：当 AI 能独立完成跨领域数学研究时，「AI 只是代码补全工具」的心智模型已严重过时。团队需重新评估 AI 在复杂推理任务（测试策略设计、系统架构决策、性能瓶颈分析）中的角色定位——不是更快的手，而是可能提供非直觉解的「第二个大脑」。

📰 科技动态

1、GitHub 确认约 3800 个内部仓库因恶意 VS Code 扩展遭窃 confirmed

GitHub 员工设备安装带毒 VS Code 扩展后，约 3800 个内部私有仓库数据被 exfiltrate；TeamPCP 黑客组织在暗网兜售源码要价 5 万美元起。公司称暂无客户仓库受影响证据，但已启动全面安全审查。

2、SpaceX 提交 S-1：2025 年营收 187 亿、合并 xAI 后 AI 占 capex 60% confirmed

SpaceX 近 400 页招股书披露 2025 年营收 187 亿美元但亏损 49 亿，Starlink 贡献过半。合并 xAI 后 AI 占资本支出约 60%，xAI 2025 年运营亏损 64 亿美元、年化 capex 约 308 亿。Musk 将保留 85%+ 投票权。

3、Railway 因 GCP 账户误封遭遇 8 小时全站 outage confirmed

Google Cloud 误将 Railway 生产账户 suspend，导致控制面、API 与 GCP 上全部工作负载中断约 8 小时。边缘路由缓存过期后连 Metal/AWS 工作负载也不可达。Railway 承认单云架构缺陷，计划将控制面移出 GCP。

4、Nvidia 季度营收 816 亿美元创纪录，私募持仓翻倍至 430 亿 confirmed

Nvidia 数据中心营收 752 亿，授权 800 亿回购。非上市股权持仓从 220 亿暴增至 430 亿。CFO 称 Blackwell 已被所有主要云厂商采用，并将为 Anthropic 显著扩容算力供给。

5、npm RFC 提议将安装脚本改为 Opt-In，从根本上缩小供应链攻击面 announced

npm 维护者提出 RFC：将 postinstall 等安装脚本从默认执行改为显式 opt-in。npm 是目前唯一默认运行依赖安装脚本的主流包管理器，该变更有望大幅降低 Shai-Hulud 类供应链攻击的有效性。

6、Firefox 148 默认关闭 asm.js，Mozilla 正式告别 OdinMonkey confirmed

SpiderMonkey 团队宣布 Firefox 148 默认禁用 asm.js JIT，未来完全移除 OdinMonkey 编译器。asm.js 曾推动 Unity/Unreal 等 C++ 代码在浏览器运行并为 WebAssembly 铺路，官方建议开发者迁移至 WASM。

5、Prime Intellect：Codex 5.5 与 Claude Opus 4.7 自主优化 nanoGPT 超越人类基线 confirmed

6、Tilde Research 发布 Aurora 优化器：修复 Muon 导致 1/4 MLP 神经元永久死亡 confirmed

Tilde 拆解 Muon 优化器发现其在 tall matrix 上导致 row-norm 各向异性，训练 warmup 阶段超过 1/4 MLP 神经元永久「死亡」。新优化器 Aurora 在 1.1B 模型上 loss 2.26 优于 Muon 2.31，MMLU 提升 10 分。

🧩 本期概念库

Agent Trace as Memory

解释：Agent 执行过程的 JSONL 记录比 PR description 或代码注释更完整地保留决策上下文。传统文档在 feature pressure 下必然 decay，而 trace 作为 byproduct 天然保持新鲜。
出处：Hugging Face Blog
重要性：Agent-native 开发中知识管理的范式转移
团队可做：将 coding agent trace 存入项目仓库，作为决策审计与新人 onboarding 的优先参考物

Important Decisions Document (IDD)

解释：在项目执行过程中持续记录重大决策的 what/why 及被否决方案的理由。源自 Stack Overflow 长期项目实践，防止反复重议旧决策并加速新人上手。
出处：ACM Queue / Thomas Limoncelli
重要性：轻量级决策管理工具，直接提升组织可靠性
团队可做：为交易系统核心模块建立 IDD，每次架构决策后 5 分钟内记录结论与被否方案

Structural Backpressure

解释：AI 编码循环应嵌入形式化验证门禁（编译、测试、lint 硬约束），用结构性反压限制 Agent 胡编，而非无限增强模型智能。验证门禁比更聪明的模型更可靠。
出处：Reuben Brooks / HN 社区
重要性：Agent 可靠性的工程原则
团队可做：在 CI 中为 Agent 生成代码增加 type-check + property-test 硬门禁，通过率作为 Agent 效果指标

Production Fork

解释：克隆生产环境做 agent 安全验证后再 collapse 回主线的部署模式。Railway CEO 断言 PR 正在死亡，Agent 时代 SDLC 从 pull request 转向 prompt + feature-flag 渐进 rollout。
出处：Railway / Latent Space
重要性：Agent-era 的 SDLC 新范式候选
团队可做：评估现有 preview 环境能否支撑 10 个并行 Agent 分支的即时部署需求
